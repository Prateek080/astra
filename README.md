# Astra

**A Claude Code plugin for the complete development lifecycle.**

`Idea` → `Spec` → `Plan` → `Implement` → `Review` → `Ship`

Cross-project. Stack-agnostic. Works in any codebase.

---

## Getting Started

### 1. Install

```bash
git clone git@github.com:Prateek080/astra.git ~/astra
echo 'alias claude="claude --plugin-dir ~/astra"' >> ~/.zshrc
source ~/.zshrc
```

Every `claude` session now loads Astra automatically.

Run `/help` inside Claude Code to verify — you should see all `/astra:*` commands.

> **For your team:** Each developer clones Astra and adds the alias. The project-specific files (`CLAUDE.md`, `.claude/rules/`) generated by `/astra:init` get committed to git — that's how the team shares project context.

---

### 2. Configure your machine *(one-time)*

```
/astra:setup
```

Interviews you about your stack, conventions, and preferences. Then configures:

- **`~/.claude/settings.json`** — effort level to high (deeper thinking) + auto-compact at 80% (prevents context from running out mid-session)
- **`~/.claude/CLAUDE.md`** — your personal coding standards, loaded into every session across all projects
- **Walks you through** `/permissions`, `/sandbox`, `/statusline` — interactive commands that can't be automated

---

### 3. Set up your project *(once per codebase)*

```
/astra:init
```

Scans your project (README, configs, directory structure), asks 2-4 clarifying questions, then generates:

- **`CLAUDE.md`** — project summary (stack, structure, dev commands, conventions). Loaded every session so Claude knows how your project works.
- **`.claude/rules/*.md`** — path-specific rules referencing real files and patterns from *your* codebase:
  - `backend.md` — *"follow the route pattern in `app/api/users.py`, use ORM models, add tests"*
  - `frontend.md` — *"reuse components from `components/ui/`, follow existing data fetching patterns"*

**You're ready to build.**

## What Command Do I Use?

Not every task needs the full workflow. Match the command to the size of the task.

```
Tiny    typo, rename, one-line fix            Just ask Claude directly.
Small   single-file fix, add a field          Just ask Claude. /astra:review after if you want.
Medium  feature across 2-5 files              plan → implement → review → ship
Large   new feature, unclear scope            spec → /clear → plan → implement → review → ship
Cleanup refactor, optimize, pay debt          debt-audit → plan → implement → review → ship
```

> All commands are prefixed with `/astra:` — for example, `/astra:plan`.

## Commands

### `/astra:spec` — Define what to build

```
/astra:spec "user auth with Google and GitHub OAuth"
/astra:spec
```

Interviews you about the feature — problem, users, scope, constraints, edge cases, non-requirements. Challenges your assumptions. Produces **`SPEC.md`**.

Next → `/clear` then `/astra:plan`

---

### `/astra:plan` — Break the work into phases

```
/astra:plan
/astra:plan "add pagination to the transactions API"
```

Explores the codebase via a subagent (keeps your context clean). Creates **`PLAN.md`** with numbered phases — each has scope, tasks, a test gate, and whether it can run in parallel.

Next → `/astra:implement`

---

### `/astra:implement` — Build phase by phase

```
/astra:implement
/astra:implement "phase 3"
```

Executes the plan one phase at a time. Searches for existing patterns before writing. Runs the test gate after each phase. Suggests `/clear` between phases. Stops and asks after 2 failed attempts.

Next → `/astra:review`

---

### `/astra:review` — Catch issues before shipping

```
/astra:review
/astra:review src/auth/
/astra:review main..HEAD
```

Reviews against a **40+ item checklist** (security, performance, quality, reusability, tech debt). Runs in a subagent. Findings grouped by severity — **Critical** / **Warning** / **Suggestion** — each with file, line, issue, and fix.

Next → Fix criticals, then `/astra:ship`

---

### `/astra:ship` — Commit and open a PR

```
/astra:ship
/astra:ship main
```

Pre-flight checks (tests, lint, secrets scan, debug artifacts), stages files explicitly, commits with a clean message, pushes, and opens a PR via `gh`. Returns the **PR URL**.

---

### `/astra:debug` — Find and fix the root cause

```
/astra:debug "TypeError: cannot read property 'id' of undefined"
/astra:debug
```

Hypothesis-driven debugging in a subagent. Finds the root cause, implements a minimal fix, writes a regression test. Reports: root cause, evidence, fix, verification, prevention.

Next → `/astra:review` then `/astra:ship`

## Additional Skills

**`/astra:debt-audit`** — Scans for dead code, orphaned TODOs, deprecated patterns, missing tests, duplication. Outputs a prioritized report.

**`/astra:workflow-guide`** — Explains which commands to use for new projects, new features, and optimizations.

**`/astra:retrospective`** — Compares spec and plan to what was actually built. Suggests improvements for your workflow.

## Context Management

> Context is the **#1 constraint** in Claude Code. As the conversation grows, older instructions get compressed or lost. When context fills up, Claude forgets your conventions and quality drops. Managing context is the single most important habit for good results.

**When to `/clear`:**
- After finishing a task, before starting something new
- Between implementation phases (phase 3 doesn't need phase 1's file reads)
- When Claude keeps making the same mistake (wrong attempts pollute context — rephrase with a fresh start)

**When to `/compact`:**
- Long session, can't clear yet — manually compress old messages around 50% context usage

**When to check `/statusline`:**
- Anytime — shows context % in your terminal. Set up once during `/astra:setup`.

**Built-in protections:** Astra commands delegate heavy exploration to subagents — their file reads happen in a separate context, not yours. Auto-compaction is set to 80% during setup (default 95% is too late).

## Under the Hood

### Agents

Commands delegate to specialized agents, each in its own context window.

| Agent | Role | Learns across projects |
|---|---|---|
| coordinator | Orchestrates agents in parallel | No |
| planner | Explores codebase, creates plans | Yes |
| implementer | Writes code, runs tests | Yes |
| reviewer | 40+ item review checklist | Yes |
| debugger | Root cause analysis and fix | Yes |

Learning agents accumulate knowledge across projects — the reviewer remembers recurring issues, the debugger remembers common root causes. Persists between sessions.

> **Coordinator mode** — for large features, run `claude --agent astra:coordinator` to spawn multiple agents in parallel.

### MCP Servers *(auto-loaded)*

| Server | Purpose |
|---|---|
| Context7 | Up-to-date library docs. Prevents hallucinated APIs. |
| Playwright | Browser automation for testing UI. |
| DeepWiki | Documentation for any GitHub repo. |

### Hooks

Desktop notifications (macOS + Linux) when Claude needs your attention.
